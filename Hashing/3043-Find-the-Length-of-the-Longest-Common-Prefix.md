# 3043. Find the Length of the Longest Common Prefix

🔗 LeetCode: https://leetcode.com/problems/find-the-length-of-the-longest-common-prefix/

---

# Problem

We are given two integer arrays `arr1` and `arr2`.

We need to find the maximum length of a common starting prefix between any number from `arr1` and any number from `arr2`.

## Example

- `100` and `1000` share prefix `"100"`
- Length = `3`

The question is actually asking:

> What is the longest digit prefix shared by any pair across the two arrays?

---

# Core Idea

A number’s prefixes can be generated by repeatedly removing the last digit.

## Example

```text
12345 → 1234 → 123 → 12 → 1
```

Key observation:

If we store all prefixes of `arr1` in a `HashSet`, then for every number in `arr2`, we can quickly check whether any of its prefixes already exists.

This avoids comparing every pair manually.

---

# Intuition

Brute force would compare every number from `arr1` with every number from `arr2`.

That becomes expensive.

Instead:

1. Precompute all prefixes from `arr1`
2. Store them in a `HashSet`
3. While processing `arr2`, check if any prefix exists in the set

Since `HashSet.Contains()` is `O(1)` average time, lookup becomes very fast.

The trick is:

Integer division by `10` naturally removes digits from the end.

## Example

```text
12345 / 10 = 1234
1234 / 10 = 123
```

So we can generate prefixes efficiently without strings.

---

# Approach

## Step 1: Store all prefixes of `arr1`

For every number:

- Keep dividing by `10`
- Store every intermediate value in a `HashSet`

## Example

```text
5678 → 5678, 567, 56, 5
```

---

## Step 2: Check prefixes of `arr2`

For every number:

- Again keep removing digits
- Check if current prefix exists in `HashSet`
- If yes, update maximum length

---

## Step 3: Return maximum length found

---

# Dry Run

## Input

```text
arr1 = [1,10,100]
arr2 = [1000]
```

---

## Prefixes stored from arr1

```text
1

100,10,1
```

## HashSet

```text
{1,10,100}
```

---

## Process 1000

```text

```

Length of `"100"` = `3`


# Pattern Used

## Hashing + Prefix Reduction

### Signals for this pattern

- Fast lookup required
- Prefix/suffix matching
- Repeated searches
- “Find common prefix”
- Avoid brute force pair comparisons

---

# Complexity Analysis

Let:

- `N = arr1.Length`
- `M = arr2.Length`
- `D = max digits in a number` (at most 10)

---

## Time Complexity

```text
O((N + M) × D)
```

### Why?

- Each number generates at most `D` prefixes
- Each `HashSet` operation is `O(1)` average

---

## Space Complexity

```text
O(N × D)
```

### Why?

We store all prefixes from `arr1`.

---

# Mistakes to Avoid

## 1. Wrong complexity calculation

Digits matter here.

Each number is processed multiple times while removing digits.

Correct complexity:

```text
O((N + M) × D)
```

NOT just:

```text
O(N + M)
```

---

## 2. Forgetting to remove digits

Need:

```csharp
curr /= 10;
```

Otherwise infinite loop.

---

## 3. Using strings unnecessarily

String conversion is only used for length calculation here.

Prefix generation itself is done numerically, which is faster.

---

## 4. Comparing all pairs

Brute force:

```text
O(N × M)
```

Too slow.

---

# Interview Tips

Interviewers usually expect:

- Recognition that prefixes can be generated mathematically
- Avoidance of brute force pair comparison
- Efficient lookup using `HashSet` or `Trie`

---

## Possible Follow-up

### “Can we solve this using Trie?”

Yes.

- Build Trie from `arr1` digits
- Traverse `arr2` digits

More scalable for advanced prefix problems.

But the `HashSet` solution is simpler and very interview-friendly.

# C# Solution

```csharp
public class Solution {
    public int LongestCommonPrefix(int[] arr1, int[] arr2) {

        HashSet<int> prefixes = new HashSet<int>();

        // Store all prefixes of arr1
        foreach (int n in arr1) {

            int curr = n;

            while (curr > 0) {
                prefixes.Add(curr);
                curr /= 10;
            }
        }

        int maxLen = 0;

        // Check prefixes of arr2
        foreach (int n in arr2) {

            int curr = n;

            while (curr > 0) {

                if (prefixes.Contains(curr)) {

                    int len = curr.ToString().Length;
                    maxLen = Math.Max(maxLen, len);
                }

                curr /= 10;
            }
        }

        return maxLen;
    }
}
```
